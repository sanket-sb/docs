## What is SFCC Integration?

SureBright Product Protection seamlessly integrates with Salesforce Commerce Cloud (SFCC) to enable merchants to offer comprehensive warranty and protection plans directly within their e-commerce storefront. This integration leverages SFCC's robust API framework and cartridge system to provide a native shopping experience for warranty products.

<Info>
  **Integration Timeline**: The complete setup typically takes 2-3 business days, depending on your SFCC environment complexity and chosen integration approach.
</Info>

## Scenario 1: Full Business Manager Access

This approach provides SureBright with comprehensive access to your SFCC environment, enabling our team to handle all technical configurations while you focus on business requirements.

<Warning>
  This approach requires granting SureBright administrative access to your SFCC Business Manager. All activities will be logged and auditable.
</Warning>

### Phase 1: SFCC User and API Configuration

The first phase involves setting up proper user access and API credentials within your SFCC Account Manager.

<Accordion title="Create SureBright User Account" iconType="duotone">
  **Purpose**: Create a dedicated SFCC user account for SureBright with appropriate permissions for integration management.

  **Step-by-Step Instructions**:
  1. **Access Account Manager**
     - Log into your SFCC Account Manager (separate from Business Manager)
     - Navigate to `Administration â†’ Users and Roles`
     - Click "Add User" button to create a new user account

  2. **Configure User Details**
     - **Username**: `surebright-integration` (must be unique)
     - **Email**: Use the email address provided by your SureBright representative
     - **First Name**: `SureBright`
     - **Last Name**: `Integration`
     - **Password**: Create a strong password (you'll share this securely with SureBright)
     - **Status**: Set to "Active" to enable immediate access

  3. **Assign Required Roles**
     In the "Roles" section, select these specific roles:
     - **Administrator**: Provides full access to Business Manager configuration, cartridge management, and system settings
     - **API User**: Enables programmatic access to SFCC APIs for data synchronization
     - **Job Manager**: Allows import/export operations and job management for data sync processes

  4. **Verify and Save**
     - Review all user details and role assignments
     - Click "Save" to create the user account
     - Note the username and password for secure sharing with SureBright

  <Warning>
    **Security Note**: This user account will have administrative access to your SFCC instance. Only share credentials through secure, encrypted channels and verify the recipient's identity before sharing.
  </Warning>
</Accordion>

<Accordion title="Configure API Client" iconType="duotone">
  **Purpose**: Create an API client that enables programmatic access to your SFCC instance for data synchronization and integration management.

  **Step-by-Step Instructions**:
  1. **Access API Settings in Account Manager**
     - In your Account Manager, navigate to `Administration â†’ API Settings`
     - Click "Create API Client" button
     - You'll need Account Manager administrative access to perform this step

  2. **Configure Client Settings**
     Fill in the following information:
     - **Client Name**: `SureBright Integration`
     - **Client Type**: Select "Public" (this allows the client to be used by external applications)
     - **Description**: `API access for SureBright Product Protection integration`
     - **Redirect URI**: Leave blank (not required for this integration type)

  3. **Assign Required OAuth Scopes**
     In the "Scopes" section, check the following boxes to grant necessary permissions:
     - `sfcc.catalogs.rw` - Read/Write access to product catalogs (allows product data sync)
     - `sfcc.products.rw` - Read/Write access to product data (enables product management)
     - `sfcc.orders.rw` - Read/Write access to order information (for warranty order processing)
     - `sfcc.custom-objects.rw` - Read/Write access to custom objects (for warranty data structures)
     - `sfcc.jobs.rw` - Read/Write access to job management (for data synchronization jobs)

  4. **Save and Retrieve Credentials**
     - Click "Save" to create the API client
     - **Important**: Copy and securely store these values immediately:
       - **Client ID**: Unique identifier for API authentication (you'll need this)
       - **Client Secret**: Secret key for API authentication (you'll need this)
       - **Organization ID**: Found in `Administration â†’ Organization â†’ Organization Settings`
       - **Realm**: Part of your SFCC URL (e.g., if URL is `https://your-realm.demandware.net`, realm is `your-realm`)
       - **Short Code**: Your site identifier (found in `Administration â†’ Sites â†’ [Your Site] â†’ Settings`)

  <Info>
    **Finding Your Organization Details**: If you're unsure about Organization ID, Realm, or Short Code, refer to the FAQ section for detailed instructions on locating these values in your SFCC Business Manager.
  </Info>
</Accordion>

<Accordion title="Share Credentials Securely" iconType="duotone">
  **Purpose**: Provide SureBright with the necessary credentials to establish secure API connectivity with your SFCC instance.

  **Required Information**:
  ```
  Client ID: [Your Client ID]
  Client Secret: [Your Client Secret]
  Organization ID: [Your Organization ID]
  Realm: [Your Realm]
  Short Code: [Your Short Code]
  ```

  **Security Requirements**:
  - Use encrypted communication channels (encrypted email or secure file sharing)
  - Never share credentials via unsecured email or chat platforms
  - Consider using a password manager or secure note-sharing service
  - Verify the recipient's identity before sharing sensitive information

  <Warning>
    These credentials provide access to your SFCC instance. Treat them as sensitive business information and ensure they are shared only with authorized SureBright personnel.
  </Warning>
</Accordion>

## Troubleshooting and Support

### Common Integration Issues

<Accordion title="SFCC Cartridge Deployment Issues" iconType="duotone">
  **Problem**: Cartridge fails to upload, activate, or function properly

  **Diagnostic Steps**:
  1. **Check User Permissions**
     - Verify SureBright user has Administrator role
     - Confirm cartridge upload permissions
     - Check Business Manager access rights

  2. **Verify Cartridge Compatibility**
     - Ensure cartridge is compatible with your SFCC version
     - Check for conflicting cartridges
     - Verify cartridge dependencies

  3. **Review Cartridge Priority**
     - Check cartridge priority settings
     - Ensure proper loading order
     - Verify no cartridge conflicts

  **Resolution**:
  - Contact SureBright support with error logs
  - Provide SFCC version and cartridge details
  - Share Business Manager configuration screenshots
</Accordion>

<Accordion title="API Authentication and Connectivity Issues" iconType="duotone">
  **Problem**: API calls fail with authentication errors or connectivity issues

  **Diagnostic Steps**:
  1. **Verify API Credentials**
     - Check Client ID and Client Secret accuracy
     - Verify Organization ID, Realm, and Short Code
     - Confirm API client is active and enabled

  2. **Check OAuth Scopes**
     - Verify all required scopes are assigned
     - Confirm scope permissions are correct
     - Check for scope changes or updates

  3. **Test API Connectivity**
     - Test API endpoints independently
     - Check network connectivity and firewall settings
     - Verify SFCC instance availability

  **Resolution**:
  - Regenerate API credentials if necessary
  - Update OAuth scopes as required
  - Contact SFCC support for connectivity issues
</Accordion>

<Accordion title="Data Synchronization Job Failures" iconType="duotone">
  **Problem**: Product or order sync jobs fail to import, run, or complete successfully

  **Diagnostic Steps**:
  1. **Check Job Configuration**
     - Verify XML format and structure
     - Check job parameters and settings
     - Confirm job permissions and access

  2. **Review Job Logs**
     - Check Business Manager job logs
     - Look for specific error messages
     - Identify failure patterns or timing

  3. **Validate Data Format**
     - Check product data format and structure
     - Verify required fields are present
     - Confirm data type compatibility

  **Resolution**:
  - Review and correct XML configuration
  - Update job parameters as needed
  - Contact SureBright support for job-specific issues
</Accordion>

<Accordion title="Storefront Display and Functionality Issues" iconType="duotone">
  **Problem**: Warranty options don't display correctly or functionality is broken

  **Diagnostic Steps**:
  1. **Check Cartridge Priority**
     - Verify SureBright cartridge priority
     - Check for conflicting cartridges
     - Confirm proper loading order

  2. **Validate Storefront Configuration**
     - Check cartridge settings and parameters
     - Verify site-specific configurations
     - Confirm template and controller setup

  3. **Test Product Configuration**
     - Verify warranty products are properly configured
     - Check product relationships and bundles
     - Confirm pricing and availability settings

  **Resolution**:
  - Adjust cartridge priority settings
  - Update storefront configurations
  - Contact SureBright support for display issues
</Accordion>

### General Integration Questions


<Accordion title="What SFCC permissions does SureBright need?" iconType="duotone">

  **Required Scopes**:
  - `sfcc.catalogs.rw` - Read/Write access to product catalogs

  - `sfcc.custom-objects.rw` - Read/Write access to custom objects

  - Cartridge is tested with common SFCC cartridges

  - Namespaced code prevents function and variable conflicts
  - Modular design allows selective feature activation
  
  **If Conflicts Occur**:
  - SureBright will adjust cartridge priority settings

  - Alternative deployment methods are available
  - Your existing functionality remains unaffected

### Product and Pricing Questions

<Accordion title="How are warranty products created in my SFCC catalog?" iconType="duotone">
  **Answer**: SureBright creates warranty products using SFCC's standard product creation process:
  
  **Product Creation Process**:
  - Warranty products are created as regular SFCC products
  - They're assigned to a dedicated warranty category
  - Custom attributes define warranty plan types and coverage
  - Products are linked to your main products via relationships
  
  **Product Configuration**:
  - SKUs follow a consistent naming convention
  - Pricing is dynamically calculated based on product value
  - Inventory is managed through your existing inventory system
  - Products are automatically assigned to your site's catalog
  
  **Merchant Control**:
  - You can modify warranty product descriptions
  - Pricing can be adjusted through your pricebook
  - Product availability can be controlled via inventory lists
  - Warranty products integrate with your existing product management workflow
</Accordion>
